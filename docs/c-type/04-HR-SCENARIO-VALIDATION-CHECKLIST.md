# HR Scenario Validation Checklist

## Structure

- [ ] 1シナリオは5タスクに収まっている
- [ ] 各タスクにタイトル、状況、資料、判断がある
- [ ] 各タスクに3つの選択肢がある
- [ ] 選択肢はすべて現実にあり得る
- [ ] 結果フィードバックが学生向けの言葉になっている

## Role Specificity

- [ ] その職種ならではの資料が出てくる
- [ ] その職種ならではの数字、制約、失敗リスクがある
- [ ] 「調整する」「相談する」だけで終わっていない
- [ ] 学生が仕事の誤解を1つ修正できる

## Non-Engineer Friendliness

- [ ] 専門用語は説明されている
- [ ] 社内略語を使いすぎていない
- [ ] 1文が長すぎない
- [ ] 正解探しではなく、判断傾向を見る設計になっている

## Privacy And Compliance

- [ ] 実在顧客名がない
- [ ] 個人名、メールアドレス、電話番号がない
- [ ] 機密数値がない
- [ ] 採用選考に使う個人評価の文言になっていない
- [ ] 差別的または不適切な表現がない

## Conversion Readiness

- [ ] AIまたは開発者がJSONへ変換しやすい見出し構造になっている
- [ ] タスク前画面、光るオブジェクト画面、選択肢画面、結果画面の項目が分かれている
- [ ] 背景画像は雰囲気指定に留め、画像内テキストに依存していない
