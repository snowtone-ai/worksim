# Webマーケター Revision Notes

今回はWebマーケター本体は全面修正しない。地方銀行と商品企画で固めたCタイプ品質ルールを使い、次回改修点を整理する。

## 現状の強み

- CPA、商談化率、フォーム項目、広告文、残予算などの具体物がある
- 短期CVとリード品質の葛藤が見える
- CPA改善だけでは成功と言えない構造がある

## 改修が必要な点

- roleCoreFrictionを明示する: 短期CV獲得、CPA効率、リード質、ブランド毀損、クライアント期待の衝突。
- misconceptionCorrectionを追加する: Webマーケターは広告を回す人ではなく、数字の良さと事業成果のズレを調整する仕事。
- meterEffectsを各choiceに追加する: lead_quality、conversion_volume、cpa_efficiency、brand_safety、hypothesis_learning、client_trust。
- 結果タイプを追加する: 短期CV獲得型、ブランド安全型、リード品質追求型、検証学習型、クライアント期待調整型。

## 具体観点

広告費:

- 残予算60万円だけでなく、月末消化圧と翌月予算への影響を入れる。
- 予算を使わない判断にも、機会損失とクライアント不信の代償を置く。

CVR / CPA / CTR / ROAS:

- CVR改善がリード質低下を招く場面を強める。
- CTRが高い広告文ほどブランド毀損や低品質CVにつながる構造を結果へ反映する。
- ROASや商談化率を使い、広告管理画面だけでは判断できない現実を出す。

短期CVとブランド毀損:

- 「無料」「今だけ」「誰でも」のような強い訴求を、短期数字とブランド毀損のトレードオフにする。
- クライアントから「数字は良いのに営業が怒っている」圧を入れる。

リード質:

- 会社規模、導入時期、部署、メール種別などのフォーム項目で質を表現する。
- 営業部から「学生や個人事業主ばかり」といった現場圧を入れる。

LP/クリエイティブ:

- LPのファーストビュー、導入事例、フォーム項目、サンクスページのどこを触るかでchoiceを分ける。
- クリエイティブ差し替えには制作リードタイムと審査落ちリスクを置く。

クライアント/上司の圧:

- クライアントはCPA改善を褒めるが、営業責任者は商談化しないと怒る構図にする。
- 上司は「月末レポートでどう説明するか」を迫る。

予算配分:

- A/B/Cへ均等配分する選択肢には、学習は進むが短期成果が読みにくい代償を置く。
- 勝ち広告へ寄せる選択肢には、短期CVは出るが悪い学習を強化するリスクを置く。

A/Bテスト:

- 検証設計、サンプルサイズ不足、期間不足を入れる。
- 「今週の数字で勝ち負けを決める」圧と、統計的には早すぎる現実をぶつける。

## 結果画面のタイプ判定

- 短期CV獲得型: conversion_volume、cpa_efficiencyが高い。lead_quality、brand_safetyを犠牲にしやすい。
- ブランド安全型: brand_safety、client_trustが高い。conversion_volumeを犠牲にしやすい。
- リード品質追求型: lead_qualityが高い。CVRとCPAを悪化させやすい。
- 検証学習型: hypothesis_learningが高い。短期成果説明が弱くなりやすい。
- クライアント期待調整型: client_trustが高い。数字の派手さより説明責任を重視する。
